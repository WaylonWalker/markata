  parsing markdown files\nwith frontmatter into Post objects. It provides parallel
  loading capabilities and\nhandles both modern Pydantic-based and legacy frontmatter
  validation.\n\n## Installation\n\nThis plugin is built-in and enabled by default
  through the 'default' plugin.\nIf you want to be explicit, you can add it to your
  list of plugins:\n\n```toml\nhooks = [\n    \"markata.plugins.load\",\n]\n```\n\n##
  Uninstallation\n\nSince this plugin is included in the default plugin set, to disable
  it you must explicitly\nadd it to the disabled_hooks list if you are using the 'default'
  plugin:\n\n```toml\ndisabled_hooks = [\n    \"markata.plugins.load\",\n]\n```\n\nNote:
  Disabling this plugin will prevent Markata from loading any markdown files.\nThis
  will effectively disable most of Markata's functionality.\n\n## Configuration\n\nConfigure
  loading behavior in your `markata.toml`:\n\n```toml\n[markata]\n# Directories containing
  markdown content\ncontent_directories = [\n    \"content\",\n    \"posts\"\n]\n\n#
  Optional: Set to true to use legacy frontmatter validation\nlegacy_frontmatter =
  false\n\n# Optional: Number of worker processes for parallel loading\nload_workers
  = 4\n```\n\n## Functionality\n\n## File Loading\n\nThe plugin:\n1. Discovers markdown
  files in content directories\n2. Loads file content and frontmatter\n3. Validates
  frontmatter against Post model\n4. Creates Post objects for further processing\n\n##
  Parallel Processing\n\nLoading is parallelized using:\n- Process pool for file reading\n-
  Configurable number of workers\n- Chunked file processing\n\n## Validation Modes\n\nSupports
  two validation approaches:\n1. Modern Pydantic-based validation (default)\n   -
  Strict type checking\n   - Automatic type coercion\n   - Detailed validation errors\n\n2.
  Legacy frontmatter validation\n   - Looser type checking\n   - Compatible with older
  content\n   - Less strict validation\n\n## Error Handling\n\nThe plugin provides:\n-
  Detailed validation error messages\n- Per-file error reporting\n- Graceful fallback
  to legacy mode\n- Optional strict validation\n\n## Registered Attributes\n\nThe
  plugin adds:\n- `articles`: List of loaded Post objects\n- `content_directories`:
  List of content source directories\n\n## Dependencies\n\nThis plugin depends on:\n-
  python-frontmatter for YAML parsing\n- pydantic for validation\n- multiprocessing

The `markata.plugins.load` plugin is responsible for loading and parsing markdown files
with frontmatter into Post objects. It provides parallel loading capabilities and
handles both modern Pydantic-based and legacy frontmatter validation.

## Installation

This plugin is built-in and enabled by default through the 'default' plugin.
If you want to be explicit, you can add it to your list of plugins:

```toml
hooks = [
    "markata.plugins.load",
]
```

## Uninstallation

Since this plugin is included in the default plugin set, to disable it you must explicitly
add it to the disabled_hooks list if you are using the 'default' plugin:

```toml
disabled_hooks = [
    "markata.plugins.load",
]
```

Note: Disabling this plugin will prevent Markata from loading any markdown files.
This will effectively disable most of Markata's functionality.

## Configuration

Configure loading behavior in your `markata.toml`:

```toml
[markata]
# Directories containing markdown content
content_directories = [
    "content",
    "posts"
]

# Optional: Set to true to use legacy frontmatter validation
legacy_frontmatter = false

# Optional: Number of worker processes for parallel loading
load_workers = 4
```

## Functionality

## File Loading

The plugin:
1. Discovers markdown files in content directories
2. Loads file content and frontmatter
3. Validates frontmatter against Post model
4. Creates Post objects for further processing

## Parallel Processing

Loading is parallelized using:
- Process pool for file reading
- Configurable number of workers
- Chunked file processing

## Validation Modes

Supports two validation approaches:
1. Modern Pydantic-based validation (default)
   - Strict type checking
   - Automatic type coercion
   - Detailed validation errors

2. Legacy frontmatter validation
   - Looser type checking
   - Compatible with older content
   - Less strict validation

## Error Handling

The plugin provides:
- Detailed validation error messages
- Per-file error reporting
- Graceful fallback to legacy mode
- Optional strict validation

## Registered Attributes

The plugin adds:
- `articles`: List of loaded Post objects
- `content_directories`: List of content source directories

## Dependencies

This plugin depends on:
- python-frontmatter for YAML parsing
- pydantic for validation
- multiprocessing for parallel loading

    def load_file_content(path: Path) -> tuple[Path, dict]:
        """Load file content without validation."""
        try:
            with open(path, "r") as f:
                raw_content = f.read()
            try:
                content = frontmatter.loads(raw_content)
            except Exception:
                content = None
            content["raw"] = raw_content
            return path, content
        except Exception:
            return path, None
